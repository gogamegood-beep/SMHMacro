# SMH_android — 상무병원 진료예약 (Android · .NET MAUI)

폰에서 직접 예약을 자동화하는 온디바이스 앱. **서버 불필요 · 비용 0원.**
WebView 에 병원 예약 페이지를 띄우고, JS 주입으로 슬롯 탐색→폼 작성→신청까지 수행하며,
**화면이 실시간으로 보이고** 완료 후 확인 화면을 캡처한다.

## 구성
```
MainPage.xaml(.cs)      UI(설정/버튼/로그/WebView/결과)
ResvSettings.cs         설정 모델 + 기기 저장(Preferences)
ReservationEngine.cs    WebView 자동화 엔진(탐색/입력/제출)
Platforms/Android/AndroidManifest.xml   인터넷 + 평문(http) 허용 등 권한
```

## 빌드 / 설치
```powershell
cd C:\Dev\SMH_android
dotnet build -f net10.0-android           # 컴파일 검증
dotnet build -t:Run -f net10.0-android    # 연결된 기기/에뮬레이터에 바로 실행
```

### 실기기 사이드로드(APK 전송)
```powershell
dotnet publish -f net10.0-android -c Release
# 산출물:
#   bin\Release\net10.0-android\com.smh.reservation-Signed.apk  (서명됨, 약 28MB)
```
- 이 APK 파일을 기기로 전송(USB/카톡/드라이브 등) → 파일 열어 설치
  - "출처를 알 수 없는 앱" 설치 허용이 필요할 수 있음
- 또는 USB 디버깅 ON 상태에서: `adb install -r com.smh.reservation-Signed.apk`
- 서명: 개인 테스트용 디버그 키로 자동 서명됨(스토어 배포용 아님)

## 동작 튜닝 (반영됨)
- **새로고침(폴링) 간격**: 매회 **1~5초 사이 랜덤**(화면에서 min/max ms 조정). 봇 신호·서버 부담 완화.
- **폼 입력 속도**: 각 칸을 **200~500ms 랜덤 간격**으로 한 칸씩 입력(사람처럼).
- **제출 직전 지연**: 0.3~1.0초 랜덤.
- **배터리 최적화 예외**: "배터리 최적화 예외 허용" 버튼 → 시스템 다이얼로그. 백그라운드 기상 안정성↑.
- **메모리**: 자동화 엔진 IDisposable(이벤트 해제), CTS dispose, 로그 길이 캡, 이전 캡처 파일 정리.

## 사용
1. 앱 실행 → 예약 정보 입력 → **설정 저장**
2. **지금 실행** = 즉시 폴링(테스트용) / **오픈시각까지 대기 후 실행** = 20:00에 시작
3. `테스트(신청하기 안 누름)` 켜고 먼저 확인 → 정상이면 해제하고 실제 예약
4. WebView 에서 폼이 채워지는 과정을 실시간으로 보고, 완료 후 하단 캡처 확인

## 동작 방식 (입력 메커니즘)
- WebView 가 병원 페이지를 직접 로드 → C# 가 `EvaluateJavaScriptAsync` 로 JS 주입
- 텍스트 칸: value 설정 + input/keyup 이벤트(붙여넣기와 유사) / 버튼·체크: 실제 click
- 같은 DOM·같은 제출 경로 → 사람이 폼 채우는 것과 기능적으로 동일

## 정시 알람 (구현됨)
- **오픈 N분 전(기본 5분 → 20:00이면 19:55)** 에 AlarmManager 정확 알람으로 앱을 깨운다.
  - 깨어나면 풀스크린 알림으로 앱이 떠서 **자동 실행 → 20:00까지 대기 후 폴링**.
  - 진료일(목/금/일/월)만 자동으로 다음 회차 재예약.
- 화면에서 "오픈 N분 전" 입력 → **알람 예약** 버튼. 해제는 **알람 해제**.
- Android 12+는 **정확 알람 권한** 필요 → **권한 설정** 버튼으로 허용 화면 이동.

## 다음 단계 / 실기기 튜닝 필요
- **배터리 최적화 예외** 등록 필요(설정 → 배터리 → 해당 앱 "제한 없음"). 안 하면 OEM이
  앱을 재워 알람이 늦거나 누락될 수 있음(삼성/샤오미 등 특히).
- 재부팅 시 알람 복원(BOOT_COMPLETED 수신기)은 권한만 추가돼 있고 수신기는 추후 구현 권장.
- 풀스크린 인텐트/백그라운드 액티비티 기동은 Android 버전·기기별 차이가 커서 **실기기 검증 필수**.
- WebView 의 `EvaluateJavaScriptAsync` 는 기기별 반환/이스케이프 차이가 있어,
  실제 오픈된 폼에서 한 번 테스트하며 셀렉터를 확정하는 것을 권장.
- 실제 '예약하기' 폼의 환자명 칸/동의 체크박스 정확한 name 미확정 → 라벨 기반 처리 중.
